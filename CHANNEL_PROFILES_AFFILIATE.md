# Канали та партнерський режим — v13

Dashboard тепер розділяє дві речі:

- **звичайний ролик** — без брендів, продажу й партнерських посилань;
- **партнерський ролик** — про конкретний схвалений AI-сервіс, із вашим
  referral URL та обов'язковою позначкою про комісію.

Кожен YouTube-акаунт має окремий профіль і окремий OAuth refresh token.
Система не використовує token основного каналу для партнерського каналу.

## Додати другий YouTube-канал

1. Відкрийте Dashboard.
2. У блоці **Профілі YouTube-каналів** введіть назву, наприклад
   `AI-сервіси UA`.
3. Натисніть **Додати канал**.
4. Google покаже вибір акаунта. Виберіть Gmail, до якого належить потрібний
   YouTube-канал, і підтвердьте доступ.
5. Після повернення на Dashboard новий канал з'явиться у полі
   **На який канал**.

Один Google OAuth Client ID достатній для всіх ваших каналів. Redirect URI
залишається тим самим. Якщо OAuth app має статус Testing, кожен Google-акаунт
треба додати до Test users у Google Auth Platform → Audience.

## Додати партнерську пропозицію

Спочатку зареєструйтеся в офіційній affiliate/referral-програмі сервісу та
отримайте особисте публічне посилання. Звичайне посилання на головну сторінку
комісію не нараховує.

У блоці **Партнерські пропозиції** заповніть:

1. канал, для якого дозволена пропозиція;
2. назву AI-сервісу;
3. ваше партнерське `https://...` посилання;
4. тільки перевірений опис функцій сервісу;
5. ключові слова й короткий CTA.

Не вставляйте API-ключі, Client Secret, refresh token або пароль. Сценарій не
вигадує ціну, функції, особистий досвід, дохід чи гарантії. Посилання й
disclosure автоматично додаються до опису ролика.

## Створити ролик

У блоці **Нова генерація** виберіть:

- канал;
- `Звичайний` або `Партнерський`;
- партнерську пропозицію, якщо потрібна;
- тематику;
- `Лише на вибраний YouTube`, `Всі платформи` або `Тільки MP4`.

Партнерська пропозиція жорстко прив'язана до профілю. Backend відхилить
запит, якщо пропозиція одного каналу випадково вибрана для іншого.

## Збереження каналів після Render redeploy

Основний старий `YOUTUBE_REFRESH_TOKEN` продовжує працювати як раніше.
Додаткові профілі зберігаються в SQLite. Щоб вони переживали redeploy,
підключіть Render Persistent Disk і задайте:

```text
DATABASE_PATH=/var/data/youtube_automation.db
```

Без persistent disk додатковий профіль працюватиме на поточному instance, але
може потребувати повторного Google-входу після нового deployment.
